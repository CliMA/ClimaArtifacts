# Pre-computed OGW Topographic Drag

Pre-computed orographic gravity wave (OGW) drag topographic tensor fields for
the ClimaAtmos cubed-sphere grid at different horizontal resolutions.

These artifacts are generated by `ClimaAtmos.compute_OGW_info` from raw
60-arcsecond ETOPO2022 topography data, following the methodology in
[Garner (2005)](https://doi.org/10.1175/JAS3496.1).

## File Naming Convention

The artifact filenames follow the pattern:
`computed_drag_Earth_{topo_smoothing}_{topography_damping_factor}_{h_elem}.hdf5`

- **`topo_smoothing`**: Whether topographic smoothing is applied (`false` in current artifacts).
- **`topography_damping_factor`**: Damping factor applied to the topography (`1` in current artifacts).

## Artifacts

| Artifact Name | h_elem | File |
|---|---|---|
| `ogw_computed_drag_h6` | 6 | `computed_drag_Earth_false_1_6.hdf5` |
| `ogw_computed_drag_h8` | 8 | `computed_drag_Earth_false_1_8.hdf5` |
| `ogw_computed_drag_h12` | 12 | `computed_drag_Earth_false_1_12.hdf5` |
| `ogw_computed_drag_h16` | 16 | `computed_drag_Earth_false_1_16.hdf5` |

## Variables

Each HDF5 file contains a single ClimaCore `Field` named `computed_drag` with
the following components on the cubed-sphere horizontal space:

- `t11`, `t12`, `t21`, `t22`: Orographic stress tensor components
- `hmax`: Maximum sub-grid topographic height
- `hmin`: Minimum sub-grid topographic height

## Usage

```julia
using ClimaUtilities.ClimaArtifacts: @clima_artifact
path = joinpath(@clima_artifact("ogw_computed_drag_h8"), "computed_drag_Earth_false_1_8.hdf5")
```

Or via the ClimaAtmos accessor:

```julia
import ClimaAtmos: AtmosArtifacts as AA
path = AA.ogw_computed_drag_file_path(; h_elem = 8)
```
